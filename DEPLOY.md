# Как запустить бота на 24/7 (не на своём Mac)

Бот делает только **исходящие** запросы (к `api-seller.ozon.ru` и
`api.telegram.org`) — ему никогда не нужен домен, открытый порт или
SSL-сертификат. Поэтому подходит любая площадка, где можно просто оставить
работать Python-процесс (или запускать его по расписанию).

Три варианта ниже — от самого надёжного (постоянная ВМ, бот работает
ровно как сейчас) до самого «бесплатного, но с оговорками» (GitHub
Actions).

---

## Вариант 1 (рекомендуется): Yandex Cloud — постоянная ВМ

Российская карта подходит, обычно при регистрации дают грант (пробные
рубли на первое время). Дальше — обычный pay-as-you-go, для такой лёгкой
задачи (бот почти не ест CPU/RAM) это будет самая дешёвая конфигурация —
точную цену покажет калькулятор при создании ВМ.

### 1. Завести ВМ
1. Регистрация на [cloud.yandex.ru](https://cloud.yandex.ru), привязать карту.
2. Консоль → **Compute Cloud** → «Создать ВМ».
3. Образ: **Ubuntu 22.04 LTS** (или 24.04).
4. Платформа/конфигурация — самая дешёвая: 2 vCPU с гарантированной долей
   5–20%, 1–2 ГБ RAM, диск 10–20 ГБ (стандартный, не SSD — незачем).
5. При создании — вставить свой SSH-публичный ключ (на Mac, если ключа
   ещё нет: `ssh-keygen -t ed25519` → возьмите `~/.ssh/id_ed25519.pub`).
6. Публичный IP — оставить включённым (проще для SSH; для самого бота он
   не нужен, только для вашего доступа).

### 2. Подключиться и подготовить сервер
```bash
ssh <ваш_логин>@<IP-адрес-ВМ>
sudo apt update && sudo apt install -y python3-venv python3-pip
```

### 3. Перенести проект
С вашего Mac (в отдельном окне терминала):
```bash
scp -r ~/Documents/ozon-order-bot <ваш_логин>@<IP-адрес-ВМ>:~/
```
(`.env` со всеми ключами уедет вместе с папкой — так и должно быть, это
единственное место, где хранятся секреты).

### 4. Установить зависимости на сервере
```bash
cd ~/ozon-order-bot
rm -rf venv                     # venv с Mac не заработает на Linux — пересоздаём
python3 -m venv venv
./venv/bin/pip install -r requirements.txt
chmod 600 .env
```

### 5. Автозапуск и автоперезапуск (systemd)
Создать `/etc/systemd/system/ozon-bot.service`:
```ini
[Unit]
Description=Ozon order bot
After=network.target

[Service]
Type=simple
WorkingDirectory=/home/<ваш_логин>/ozon-order-bot
ExecStart=/home/<ваш_логин>/ozon-order-bot/venv/bin/python bot.py
Restart=always
RestartSec=5
User=<ваш_логин>

[Install]
WantedBy=multi-user.target
```
Затем:
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now ozon-bot
sudo systemctl status ozon-bot      # должен быть "active (running)"
journalctl -u ozon-bot -f            # логи в реальном времени, Ctrl+C для выхода
```

Теперь бот работает ровно как на вашем Mac — тот же непрерывный процесс,
только на сервере, который никогда не спит. Перезагрузка ВМ / падение
процесса — `systemd` сам перезапустит.

### Обновление кода в будущем
```bash
scp -r ~/Documents/ozon-order-bot/*.py <ваш_логин>@<IP-адрес-ВМ>:~/ozon-order-bot/
ssh <ваш_логин>@<IP-адрес-ВМ> "sudo systemctl restart ozon-bot"
```

### VK Cloud — то же самое
Если Yandex Cloud не подойдёт — [VK Cloud](https://mcs.mail.ru) принимает
российские карты и работает по тому же принципу (ВМ + systemd). Разница
только в интерфейсе консоли при создании ВМ, шаги 2–5 выше — идентичны.

## Вариант 2: Google Cloud Always Free (вероятно не получится зарегистрироваться)

**Важная оговорка:** Google Cloud при регистрации требует привязать карту
(технический холд ~$1). Российские карты Visa/Mastercard с 2022 года не
обслуживают международные платежи — скорее всего, привязка не пройдёт без
не-российской карты. Если получится — дальше всё просто и действительно
бесплатно навсегда, в рамках лимитов.

1. Регистрация на [cloud.google.com](https://cloud.google.com), привязать
   карту, включить биллинг (в рамках Always Free счёт не придёт, если не
   выходить за лимиты ниже).
2. Консоль → **Compute Engine** → включить API, если попросит.
3. «Создать инстанс»:
   - **Machine type: `e2-micro`** — обязательно именно эта, только она
     входит в Always Free.
   - **Region: строго одна из** `us-west1`, `us-central1`, `us-east1` —
     иначе бесплатный лимит не действует и придёт счёт.
   - Boot disk: Ubuntu 22.04 LTS, до 30 ГБ стандартного диска (входит в
     лимит).
4. Файрвол — ничего менять не нужно (боту не нужны входящие соединения).
5. Подключение — кнопка **SSH** прямо в браузере, в консоли рядом с
   инстансом (свой ключ не нужен).
6. Дальше — те же шаги 2–5 из варианта Yandex Cloud выше (`apt install
   python3-venv`, перенос проекта, venv, systemd-юнит).

Лимиты Always Free (актуальны на момент написания, могут измениться —
стоит проверить на странице Google Cloud Free Tier): 1 инстанс `e2-micro`
непрерывно, 30 ГБ диска, 1 ГБ исходящего трафika в месяц — для этого бота
с огромным запасом (он гоняет мелкие JSON-запросы и изредка PDF).

## Вариант 3: GitHub Actions (бесплатно, но с реальными неудобствами)

**Перед тем как выбрать этот вариант — оговорки:**

- Бот перестанет работать непрерывно. Минимальный интервал у GitHub
  Actions — **раз в 5 минут** (`cron: '*/5 * * * *'`), и по факту на
  бесплatных аккаунтах в часы нагрузки запуск может опаздывать на
  10–15 минут. Сейчас бот опрашивает Ozon раз в 90 секунд — с GitHub
  Actions уведомления и этикетки будут приходить заметно позже.
- Кнопки (**🖨 Печать**, поиск, «отметить/снять все») перестанут быть
  мгновенными — нажатие кнопки обработается только на следующем запуске
  по расписанию, то есть тоже с задержкой до 5–15 минут между нажатием и
  ответом. Для интерфейса с кнопками это ощутимо неудобно на практике.
- Между запусками нет постоянного диска — состояние (какие заказы уже
  отправлены/напечатаны, то, что гарантирует «без дублей») придётся
  сохранять через `actions/cache`. Это лучшее из простых решений, но кэш
  иногда может очиститься сам — тогда часть уведомлений задублируется.

Если эти компромиссы устраивают — вот что нужно:

### 1. Код: режим «один проход и выход»
Сейчас `bot.py` — это бесконечный процесс (поллинг раз в 90 сек + вечный
long-polling Telegram). Для GitHub Actions нужен **отдельный скрипт**
`run_once.py`, который один раз вызывает то же самое (`poll_once()` +
одна проверка команд Telegram через `getUpdates` с `timeout=0`) и
завершается — я допишу его, если решите пойти этим путём (небольшое,
самостоятельное дополнение, не трогает `bot.py`).

### 2. Репозиторий и секреты
1. `git init`, закоммитить код бота (`.env` не коммитить — он уже в
   `.gitignore`). Репозиторий может быть **приватным** (2000 минут
   Actions в месяц бесплатно — этому боту с большим запасом хватит) или
   публичным (лимит минут не действует вовсе, но код будет виден всем —
   секретов в коде и так нет, только в GitHub Secrets, так что публичный
   репозиторий не течёт ключами, но решать вам).
2. На GitHub: **Settings → Secrets and variables → Actions → New
   repository secret** — добавить по одному секрету на каждую переменную
   из `.env`: `SHOP_1_CLIENT_ID`, `SHOP_1_API_KEY`, `SHOP_2_NAME`, ...,
   `BOT_TOKEN`, `CHAT_ID`.

### 3. Файл `.github/workflows/poll.yml`
```yaml
name: Ozon bot poll

on:
  schedule:
    - cron: '*/10 * * * *'   # раз в 10 минут (можно поставить */5)
  workflow_dispatch: {}        # + можно запустить вручную кнопкой на GitHub

jobs:
  poll:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - name: Восстановить состояние (какие заказы уже обработаны)
        uses: actions/cache@v4
        with:
          path: bot_state.sqlite3
          key: ozon-bot-state
      - run: pip install -r requirements.txt
      - run: python run_once.py
        env:
          SHOP_1_NAME: ${{ secrets.SHOP_1_NAME }}
          SHOP_1_CLIENT_ID: ${{ secrets.SHOP_1_CLIENT_ID }}
          SHOP_1_API_KEY: ${{ secrets.SHOP_1_API_KEY }}
          SHOP_2_NAME: ${{ secrets.SHOP_2_NAME }}
          SHOP_2_CLIENT_ID: ${{ secrets.SHOP_2_CLIENT_ID }}
          SHOP_2_API_KEY: ${{ secrets.SHOP_2_API_KEY }}
          SHOP_3_NAME: ${{ secrets.SHOP_3_NAME }}
          SHOP_3_CLIENT_ID: ${{ secrets.SHOP_3_CLIENT_ID }}
          SHOP_3_API_KEY: ${{ secrets.SHOP_3_API_KEY }}
          BOT_TOKEN: ${{ secrets.BOT_TOKEN }}
          CHAT_ID: ${{ secrets.CHAT_ID }}
      - name: Сохранить состояние
        uses: actions/cache/save@v4
        with:
          path: bot_state.sqlite3
          key: ozon-bot-state
```

Скажите, если хотите пойти этим путём — допишу `run_once.py` и можно будет
запускать прямо отсюда.

## Вкратце

| Вариант | Стоимость | Как бот себя ведёт | Сложность |
|---|---|---|---|
| Yandex/VK Cloud | ~недорого, есть грант на старт | как сейчас, без изменений | ВМ + systemd, ~20 минут один раз |
| Google Cloud Always Free | бесплатно навсегда, **если получится зарегистрироваться** | как сейчас, без изменений | то же самое,+ проблема с картой |
| GitHub Actions | бесплатно | уведомления и кнопки с задержкой 5–15 мин, риск редких дублей | нужен доп. код (`run_once.py`) |
